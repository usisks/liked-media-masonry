# 模擬ストレステスト報告

対象バージョン：0.14.4  
実施日：2026年7月13日

## 目的

実際のXへ接続せず、ローカルで生成した投稿DOMを使い、大量収集時の索引整合性、差分追加、ライトボックス維持、Observer再接続、動画要素保持、カード解放、DOM数、JavaScriptヒープを確認する。

この結果はXの通信、実DOM、実動画配信、Chrome通常プロファイルでの30分利用を再現するものではなく、実X手動回帰テストの代替ではない。

## 実行方法

```bash
npm run test:stress
```

使用条件：

- Headless Chromium
- 画面サイズ 1440×900
- X画像・動画への通信は遮断
- Chrome APIはローカルモック
- content scriptはManifestと同じ順序で読み込み
- CDP Performance MetricsからJavaScriptヒープを取得

## シナリオA：段階追加・DOM交換

実施内容：

1. 画像投稿600件を一括走査
2. 100件ずつ5回追加
3. 追加中も中央のライトボックス項目を維持
4. 同じDOMを再走査して重複が増えないことを確認
5. ボードを上下20往復
6. ライトボックスを40回開閉
7. X側タイムラインを別のsectionへ交換
8. 新しいタイムライン50件を走査
9. MutationObserver経由で25件を後挿入
10. blob動画要素80件を交換し、強参照上限を確認
11. ページ離脱相当の停止処理を実行

結果：

| 項目 | 結果 |
|---|---:|
| 最初の600件走査 | 25.1ms |
| 100件追加・1回目 | 48.9ms |
| 100件追加・2～5回目 | 14.5～16.5ms |
| 重複再走査 | 81.4ms、追加0件 |
| 最終収集件数 | 1,175件 |
| `itemMap` | 1,175件 |
| `itemIndexMap` | 1,175件 |
| `mediaPathIndex` | 1,175件 |
| カード外枠 | 1,175件 |
| hydratedカード | 1件 |
| DOM要素数 | 1,569 |
| 強参照video | 3件 |
| 非表示保管video | 0件 |
| JavaScriptヒープ使用量 | 3,742,748 bytes |
| ページエラー | 0件 |

確認事項：

- 段階追加後もライトボックスの表示中メディアを維持
- タイムライン交換後にObserverが新しいrootへ接続
- 拡張UIのoverlay、launcher、lightboxは各1個だけ
- カード解放キューは待機後に0件
- 停止処理後はObserver、強参照video、LoadControllerを解放

## シナリオB：5,000件規模

実施内容：

1. 画像投稿5,000件を一括走査
2. 元タイムラインDOMを除去し、X側の仮想化後に近い状態へする
3. ボードを開く
4. 同じDOMを再走査
5. 最下部へ移動し、カード解放キューの収束を確認

結果：

| 項目 | 結果 |
|---|---:|
| 5,000件走査 | 260.8ms |
| 再走査 | 11.6ms、追加0件 |
| 最終収集件数 | 5,000件 |
| 3索引の件数 | 各5,000件 |
| カード外枠 | 5,000件 |
| hydratedカード | 28件 |
| プレースホルダー | 4,972件 |
| DOM要素数 | 5,188 |
| JavaScriptヒープ使用量 | 13,079,288 bytes |
| ページエラー | 0件 |

## 判定

模擬ストレステストは通過した。

- 同一投稿・同一メディアの重複増加なし
- 収集配列と3索引の件数一致
- 画面付近だけをhydrate
- video強参照は最大3件
- カード解放キューの残留なし
- SPA相当のタイムライン交換後も再接続
- 反復開閉で重複UIなし

## 残る制限

- 5,000件時もカード外枠は5,000件分DOMへ残る。完全仮想スクロールは未実装。
- 測定値はこの実行環境の単回結果であり、一般的な性能保証値ではない。
- X本体の投稿仮想化、通信遅延、広告、警告画面、動画配信方式は再現していない。
- 実際のXで500件以上を読み込むX-34と、30分以上利用するX-35は未確認のまま。
