# Chromeウェブストア掲載文案（日本語）

対象バージョン：0.14.4

この文書はDeveloper Dashboardへの入力用草案です。公開前に、実際のX上での手動回帰テスト、サポート先、プライバシーポリシー公開URL、掲載画像を確定してください。

## 名称

Liked Media Masonry

## 短い説明

Xのいいね欄に読み込まれた画像・動画・GIFを、可変高さのボードと同一画面の拡大ビューで閲覧できます。

## 単一目的

Xの「いいね」ページに利用者のブラウザがすでに読み込んだメディアを、見やすい可変高さボードへ整理し、同じ画面で順番に閲覧・再生できるようにすること。

## 詳細説明

Liked Media Masonryは、Xの「いいね」ページに読み込まれた画像・動画・GIFを、投稿本文を表示しない可変高さのボードへ整理するChrome拡張機能です。

主な機能：

- 縦長画像を切らず、元の縦横比を保ったボード表示
- 画像・動画・GIFを同一画面の拡大ビューで表示
- 左右キーと左右プレビューによる前後移動
- 拡大表示中のホイール・上下キーによる背景ボードのスクロール
- ボード最下部付近からの追加読込
- カード幅、左右プレビュー透明度、動画表示、拡大表示終了後の位置を設定可能
- 診断情報は利用者が操作した場合だけクリップボードへコピーし、自動送信しない

本拡張機能はXまたはPinterestの公式製品ではなく、両社から承認・提供・推奨されたものでもありません。Xの画面構造をブラウザ内で読み取る方式のため、X側の変更によって一部の取得や再生が動作しなくなる場合があります。

## 権限とサイトアクセスの説明

### storage

カード幅、動画表示、拡大表示を閉じた後の位置、左右プレビュー透明度の4設定を`chrome.storage.local`へ保存するために使用します。

### x.com / twitter.comへのサイトアクセス

Xの「いいね」ページに利用者のブラウザがすでに表示した投稿DOMから、画像・動画サムネイル・投稿URL・表示名・投稿日を読み取り、ボード表示へ変換するために使用します。

対象は`https://x.com/*`と旧URL互換用の`https://twitter.com/*`だけです。全サイトへのアクセス権限は要求しません。

## データ利用の要約

- 投稿、いいね履歴、画像URL、動画URL、アカウント情報を開発者のサーバーへ送信しません。
- 広告、アクセス解析、追跡、販売、第三者提供を行いません。
- パスワード、Cookie、セッショントークンを取得しません。
- 永続保存するのは表示設定4項目だけです。
- 診断情報は固定許可リストから作成し、利用者が明示的にコピーした場合だけクリップボードへ保存します。

詳細は公開URLへ配置した`PRIVACY.md`と同内容のプライバシーポリシーを参照してください。

## ストア用画像計画

### スクリーンショット

1280×800を推奨し、少なくとも1枚、可能なら次の5枚を用意します。

1. 画像を可変高さで並べたボード全体
2. 縦長画像が切れずに表示されている状態
3. ライトボックスと左右プレビュー
4. 動画またはGIFの再生画面
5. ポップアップの表示設定

撮影には自分で公開した投稿または掲載許可を得たテスト用投稿を使用し、私的ないいね履歴、第三者の非公開情報、不要なユーザー名を映さないでください。

### 小型プロモーション画像

- 440×280
- スクリーンショットの流用ではなく、ボード表示を想起できる簡潔なブランド画像
- XまたはPinterest公式と誤認させるロゴ・表現を使用しない

### アイコン

ZIPには128×128 PNGを含めています。現行アイコンは透明余白がないため、公開前にストアの推奨余白へ合わせた版を作るか判断してください。

## Developer Dashboardで確定が必要な項目

- 公開プライバシーポリシーURL
- サポートURLまたは問い合わせ先
- 開発者表示名と連絡先
- カテゴリ
- 言語と地域
- スクリーンショット1～5枚
- 440×280のプロモーション画像
- 単一目的の説明
- データ利用申告
- 公開範囲
