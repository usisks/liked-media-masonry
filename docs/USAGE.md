# 使い方

## ボードを開く

1. Xへログインし、自分の「いいね」ページを開きます。
2. 閲覧したい範囲まで通常のX画面を読み込みます。
3. 右下の「いいねビューに変える」、またはChromeツールバーのLiked Media Masonryアイコンから「ボードを開く」を選びます。

ボード表示中は起動ボタンが左上の「Xの表示に戻る」に変わります。

## 画像・動画・GIF

- カードを選ぶとライトボックスで拡大表示します。
- 画像は元のX投稿へのリンクです。
- 動画・GIFは、Xの画面が保持している再生情報を利用できる場合に再生します。
- 再生情報がない場合はサムネイルと「Xで再生する」リンクを表示します。

## ライトボックス

- `←` / `→`: 前後のメディアへ移動
- `↑` / `↓`: 背景ボードをスクロール
- `Page Up` / `Page Down`: 背景ボードを大きくスクロール
- `Space`: 背景ボードを下へスクロール
- `Esc`: ライトボックスを閉じる

最初と最後を越えて先頭・末尾へループしません。中央画像はTabで選択し、Enterで元投稿を新しいタブに開けます。

## 背景ボードのスクロール

ライトボックス表示中も、マウスホイールとキーボードで背景ボードを操作できます。ライトボックスを閉じた後の位置は、設定で「現在位置を維持」または「開く前へ戻る」から選択できます。

## 追加読込

ボード下端へ近づくと、利用者の直前操作がある場合にXのページを下へ進め、新しく表示されたメディアを取り込みます。

- ポップアップの「さらに読み込む」で手動実行できます。
- 「画面内を再走査」で現在のDOMをもう一度確認できます。
- 重複メディアは追加しません。
- 新規メディアが続けて見つからない場合は、自動読込を一時休止します。

## 設定

- カード幅
- 左右プレビューの透明度
- 動画・GIFを表示するか
- ライトボックスを閉じた後の位置

動画・GIFをオフからオンへ変更すると、現在の収集結果を破棄し、Xのページ先頭からボードを作り直します。設定は `chrome.storage.local` に保存されます。

## 問題が発生した場合

1. Xと拡張機能を再読み込みします。
2. 「画面内を再走査」を実行します。
3. [既知の制限](KNOWN_LIMITATIONS.md) を確認します。
4. 必要に応じて「エラー報告をコピー」を実行し、個人情報が含まれていないことを確認してIssueへ添付します。
