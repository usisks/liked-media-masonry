# セキュリティポリシー

## 脆弱性の報告

脆弱性、認証情報の漏えい、利用者データの意図しない外部送信が疑われる場合は、公開Issueへ詳細を書かないでください。GitHubリポジトリの「Security」から「Report a vulnerability」を選び、Private Vulnerability Reportingを利用してください。

報告には、影響するバージョン、再現手順、想定される影響、確認に必要な最小限の情報を含めてください。Cookie、セッショントークン、非公開投稿の全文など、実データの添付は避けてください。

## サポート対象

最新のGitHub Releaseのみをセキュリティ修正の対象とします。古いバージョンで問題を確認した場合は、最新バージョンでも再現するかを可能な範囲で確認してください。

## 対応できない内容

- X自体の脆弱性やアカウント問題
- ChromeまたはOS自体の脆弱性
- 改変された非公式ビルド
- 対象サービスの利用規約や著作権に関する個別の法的判断

一般的な不具合や機能要望は [SUPPORT.md](SUPPORT.md) に従ってIssueへ報告してください。
