# Liked Media Masonry

Xの「いいね」ページに読み込まれた画像・動画・GIFを抽出し、可変高さのMasonryボードと拡大ビューで閲覧するManifest V3対応ブラウザ拡張機能です。Chrome系ブラウザに加えて、FirefoxとFloorpへFirefox WebExtensions互換版を提供します。

本拡張機能はXまたはPinterestの公式製品ではなく、各社から承認、提供、推奨されたものではありません。

## 主な機能

- Xの「いいね」ページに表示された画像、動画、GIFを収集
- 縦横比を保ったMasonryボード表示
- 前後プレビュー付きライトボックス
- 画像・動画・GIFを同じ画面で閲覧
- カード幅、左右プレビュー透明度、動画表示、閉じた後の位置を設定
- ページ下端からの追加読込と手動再走査
- 投稿者名、ユーザーID、投稿日、元投稿へのリンクを表示
- 設定をブラウザ拡張機能の `storage.local` に保存
- 個人情報を除外した診断情報を、利用者の操作でクリップボードへコピー

## 対応ページ

- `https://x.com/*`
- `https://twitter.com/*`

主な利用場所は、利用者自身が閲覧できるXの「いいね」ページです。ログイン状態、表示形式、X側のDOM変更によって動作しない場合があります。

## ダウンロードとインストール

Chrome、Edge、Brave、Vivaldi、Operaでは、GitHub Releasesの `liked-media-masonry-chrome-v0.15.0.zip` を展開し、拡張機能のデベロッパーモードから読み込みます。

Firefox 140以降と対応するFloorpでは、同じReleaseにある `liked-media-masonry-firefox-floorp-v0.15.0.xpi` を開いてインストールします。このXPIはMozillaのAMO unlistedチャネルで署名された自己配布版です。`-unsigned.xpi` と付く開発用ファイルやChrome ZIPをFirefoxへインストールしないでください。

Firefox・Floorp版は初回インストール後、固定されたGitHub Pagesの更新マニフェストを確認し、新しいMozilla署名済みXPIへ更新できます。手動確認は `about:addons` の歯車メニューから「更新を確認」を選びます。

詳しい手順と更新・削除方法は [インストールガイド](docs/INSTALL.md) を参照してください。ChromeウェブストアおよびAMO listed公開版ではありません。

## 基本的な使い方

1. Xで自分の「いいね」ページを開き、表示したい範囲を読み込みます。
2. 画面右下の「いいねビューに変える」、またはChromeツールバーの拡張機能アイコンからボードを開きます。
3. カードを選ぶとライトボックスが開きます。矢印キーで前後移動し、Escで閉じます。
4. ボード下端では追加読込が行われます。ポップアップから再走査や手動追加読込も実行できます。

操作の詳細は [使い方](docs/USAGE.md) を参照してください。

## 設定

- カード幅: 180〜640px、または小・標準・大のプリセット
- 左右プレビューの透明度: 0〜90%
- 動画・GIFの表示
- ライトボックスを閉じた後のスクロール位置

設定はブラウザ拡張機能の `storage.local` に保存します。`localStorage` は使用しません。

## 権限

要求する拡張機能権限は `storage` のみです。表示設定を端末内へ保存するために使用します。

content scriptの対象は `x.com` と `twitter.com` に限定しています。対象ページのDOMから、既に表示または読み込まれたメディア情報を取得するために必要です。外部APIへの送信には使用しません。詳細は [権限の説明](docs/PERMISSIONS.md) を参照してください。

## プライバシー

投稿、メディアURL、いいね履歴、アカウント情報、診断情報を開発者または第三者のサーバーへ送信しません。Cookie、パスワード、セッショントークンは取得・保存しません。診断情報のコピーは利用者が明示的に実行した場合だけ行われ、自動送信されません。

詳細は [プライバシーポリシー](PRIVACY.md) を参照してください。

## 既知の制限

- XのDOMや動画配信方式が変更されると、抽出や再生が失敗する場合があります。
- 画面へ読み込まれていない投稿やメディアは取得できません。
- `blob:` 動画などは、X側が動画要素を破棄した後には拡張機能内で再生できない場合があります。
- 大量表示ではカード外枠がDOMに残るため、件数に応じて負荷が増えます。
- 実際のX上での全環境・全表示形式を自動テストしているわけではありません。

詳しくは [既知の制限](docs/KNOWN_LIMITATIONS.md) を参照してください。
